# Create a YAML Document

Main outputs of an Anemos project are YAML documents. These documents define the Kubernetes manifests
that will be applied to the cluster.

Each `Document` has a file path that defines where it will be saved in the output directory. The file path
is relative to the output directory and can contain subdirectories.

The following example shows various ways to create YAML documents.
